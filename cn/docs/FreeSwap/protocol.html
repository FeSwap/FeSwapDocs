FreeSwap 交易协议 #

交易协议目标 #

FreeSwap 交易协议目标实现一个完全没有交易手续费的去中心化交易所。"恒定资产乘积" 交易机制不可避免地产生交易价格滑动，FreeSwap 协议通过锁定一定幅度的价格滑动，并从价格滑动中自动套利，为流动性提供者提供收益来源。

交易对设置 #

FreeSwap 交易协议为每对数字资产 (TokenA∣TokenB)(Token_A|Token_B)(TokenA​∣TokenB​) 设置两个独立的单向子交易池，表示为：

(NAA ∣ NB) ∣∣ (NA ∣ NBB)(2.2.1)( N_{AA}\ |\ N_B)\ || \ (N_A\ |\ N_{BB} ) \tag{2.2.1}(NAA​ ∣ NB​) ∣∣ (NA​ ∣ NBB​)(2.2.1)

其中, 子交易对 (NAA ∣ NB)(N_{AA}\ |\ N_B )(NAA​ ∣ NB​) ( 简称 AAA 子交易对 、AAA 池 ) 可以单向兑入 TokenAToken ATokenA 、兑出 TokenBToken BTokenB，NAAN_{AA}NAA​ 为 AAA 池内 TokenAToken ATokenA 的数量，NBN_BNB​ 为 AAA 池内 TokenBToken BTokenB 的数量。

子交易对 (NA ∣ NBB)(N_A\ |\ N_{BB} )(NA​ ∣ NBB​) ( 简称 BBB 子交易对 、BBB 池 ) 可以单向兑入 TokenBToken BTokenB 、兑出 TokenAToken ATokenA，NBBN_{BB}NBB​ 为 BBB 池内 TokenBToken BTokenB 的数量，NAN_ANA​ 为 BBB 池内 TokenAToken ATokenA 的数量。

流动性提供者向交易池注入资金时，可以指定所要注入的子交易池，也可以设定一定的比例，将资金同时注入两个子交易池。

兑换交易 #

用户跟交易池进行 TokenAToken ATokenA 与 TokenBToken BTokenB 的兑换交易时，会有2种相反的交易操作，一是兑入 TokenAToken ATokenA ，兑出 TokenBToken BTokenB ；一是兑入 TokenBToken BTokenB ，兑出 TokenAToken ATokenA 。这里 “兑入” 和 “兑出” 是相对交易池而言的。

如果用户兑入 TokenAToken ATokenA 、兑出 TokenBToken BTokenB ，FreeSwap 会将该交易交给 AAA 池, 即 (NAA∣NB)(N_{AA}|N_B )(NAA​∣NB​) 子交易对完成。AAA 池总是兑入 TokenAToken ATokenA ，兑出 TokenBToken BTokenB ，所以 NAAN_{AA}NAA​ 的数量一直会上升，NBN_BNB​ 的数量一直会下降，这会导致 AAA 池内以 TokenBToken BTokenB 计价的 TokenAToken ATokenA 的价格会单调下降，从而导致 TokenAToken ATokenA 的价格偏离实际市场价格，这种价格偏离可以为 FreeSwap 交易对提供套利空间。

同样，如果用户兑入 TokenBToken BTokenB ，兑出 TokenAToken ATokenA ，会由 BBB 池来完成交易。BBB 池总是兑入 TokenBToken BTokenB ，兑出 TokenAToken ATokenA ，所以 NBBN_{BB}NBB​ 的数量会一直上升，NAN_ANA​ 的数量会一直下降，这同样会导致 BBB 池内以 TokenAToken ATokenA 计价的 TokenBToken BTokenB 的价格单调下降，最终偏离实际市场价格。

子交易对 (NAA∣NB)(N_{AA} | N_B )(NAA​∣NB​)，(NA∣NBB)(N_A | N_{BB} )(NA​∣NBB​) 是两个单向的方向相反的交易对，它们总是固定兑入一种代币，兑出另一种代币。随着交易的发生，两种代币的价格在两个子交易对内总是以相反的方向滑动，积累一定交易量后，同一代币在两个子交易对内的价格偏差就会超过一定的幅度。价差过大会阻止用户继续参与兑换交易，这时需要在两个子交易对内进行两种资产的资金调配，让代币的兑换价差得到恢复。

在两个子交易对内进行资金调配的过程实际上是一个交易套利过程。交易用户在两个单向交易池内进行代币交易，造成代币价格单向滑动，用户会因此蒙受价格滑动造成的交易损失。通过两个子交易池相互进行资产调配，既可以对价格滑动进行修复，又可以实现对用户交易价格滑动损失的套利，为交易池流动性提供者带来套利收益。

交易对套利 #

如上所述，FreeSwap 设置两个独立的子交易对：
(NAA ∣ NB) ∣∣ ( NA∣ NBB)(2.4.1)( N_{AA}\ |\ N_B )\ || \ (\ N_A |\ N_{BB} ) \tag{2.4.1}(NAA​ ∣ NB​) ∣∣ ( NA​∣ NBB​)(2.4.1)

TokenAToken ATokenA 、TokenBToken BTokenB 在两个子交易对内的价格随着交易发生会产生偏离，当价格偏离较大时，需要对两个子交易对进行资产调配，恢复资产价格的一致性。两个子交易对之间的资产调配既可以恢复双方的资产价格，也可以实现相互之间的套利。下面对 FreeSwap 协议的套利方法进行详细描述。

套利前，TokenAToken ATokenA 在两个交易对内以 TokenBToken BTokenB 计价的价格为：

PAA→B=NBNAA(2.4.2)P_{AA \rightarrow B} = \frac{N_B}{N_{AA}} \tag{2.4.2}PAA→B​=NAA​NB​​(2.4.2)
PA←BB=NBBNA(2.4.3)P_{A \leftarrow {BB}} = \frac{N_{BB}}{N_A} \tag{2.4.3}PA←BB​=NA​NBB​​(2.4.3)

其偏离比例为：

RPA=PA←BBPAA→B=NAA∗NBBNB ∗ NA(2.4.4)R_{PA} = \frac{P_{A \leftarrow BB}}{P_{AA \rightarrow B}} = \frac{N_{AA} * N_{BB}}{N_B \ * \ N_A} \tag{2.4.4}RPA​=PAA→B​PA←BB​​=NB​ ∗ NA​NAA​∗NBB​​(2.4.4)

同理，TokenBToken BTokenB 在两个交易对内以 TokenAToken ATokenA 计价的价格为：

PBB→A=NANBB(2.4.5)P_{BB \rightarrow A} = \frac{N_A}{N_{BB}} \tag{2.4.5}PBB→A​=NBB​NA​​(2.4.5)

其价格偏离比例为：

RPB=PB←AAPBB→A=NAA∗NBBNB∗NA(2.4.7)R_{PB} = \frac{P_{B \leftarrow AA}}{P_{BB \rightarrow A}} = \frac{N_{AA} * N_{BB}}{N_B * N_A} \tag{2.4.7}RPB​=PBB→A​PB←AA​​=NB​∗NA​NAA​∗NBB​​(2.4.7)

容易理解，两个子交易对内 TokenAToken ATokenA，TokenBToken BTokenB 的价格偏离比例相同，即：

RPA=RPB=RP=NAA∗NBBNB∗NA(2.4.8)R_{PA} = R_{PB} = R_P = \frac{N_{AA} * N_{BB}}{N_B * N_A} \tag{2.4.8}RPA​=RPB​=RP​=NB​∗NA​NAA​∗NBB​​(2.4.8)

FreeSwap 协议设定当两个子交易对的资产价格偏离，即 RPR_{P}RP​ 的值大于一定门限值 γ\gammaγ，如101% 时 ，自动启动内部套利机制，恢复价格。

套利操作就是利用一个子交易对内数量较多的资产，等价值兑换另一个子交易对内数量较多的另一个资产，即 (NAA∣NB)( N_{AA} | N_B )(NAA​∣NB​) 子交易对利用 TokenAToken ATokenA 换取 (NA∣NBB)( N_A | N_{BB} )(NA​∣NBB​) 子交易对等价值的 TokenBToken BTokenB 。换个角度看，等价于 (NA∣NBB)( N_A | N_{BB} )(NA​∣NBB​) 子交易对利用 TokenBToken BTokenB 换取 (NAA∣NB)( N_{AA} | N_B )(NAA​∣NB​) 子交易对等价值的TokenAToken ATokenA 。

为保证兑换公平，FreeSwap 协议规定，兑换价格为两个子交易对所有资产的平均价格。交换价格可表示为：

PA→Be=NB+NBBNA+NAA(2.4.9)P_{A \rightarrow B}^e = \frac{N_B + N_{BB}}{N_A + N_{AA}} \tag{2.4.9}PA→Be​=NA​+NAA​NB​+NBB​​(2.4.9)
PB→Ae= NA+NAANB+NBB(2.4.10)P_{B \rightarrow A}^e = \ \frac{N_A + N_{AA}}{N_B + N_{BB}} \tag{2.4.10}PB→Ae​= NB​+NBB​NA​+NAA​​(2.4.10)
PA→Be∗PB→Ae= 1(2.4.11)P_{A \rightarrow B}^e * P_{B \rightarrow A}^e = \ 1 \tag{2.4.11}PA→Be​∗PB→Ae​= 1(2.4.11)

(NAA−LA ∣ NB+LB) ∣∣ (NA+LA ∣ NBB−LB)(2.4.12)(N_{AA} - L_A \ | \ N_B + L_B )\ ||\ ( N_A + L_A \ | \ N_{BB} - L_B ) \tag{2.4.12}(NAA​−LA​ ∣ NB​+LB​) ∣∣ (NA​+LA​ ∣ NBB​−LB​)(2.4.12)

为减少套利操作频次，子资金池之间的套利操作应当尽可能多地交换两种资产以最大幅度地恢复资产价格偏离。

不难发现，当 LAL_ALA​, LBL_BLB​ 取值上述关系式中的最大值时，两个子交易对的资产价格恰好同时恢复到这个交易池的资产平均价格 PA→BeP_{A \rightarrow B}^ePA→Be​，同时也是双方交换资产的价格。
